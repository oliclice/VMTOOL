# CLAUDE.md - VM-TOOL 现代化重构项目

## 项目概述

**码表处理工具**正在经历一次全面的现代化重构，目标是从一个简单的脚本工具升级为工业级、高性能、可扩展的码表管理系统。

**当前状态**: 重构设计阶段已完成，实施计划制定中
**目标版本**: 2.0.0
**技术栈**: FastAPI + SQLAlchemy + Typer + SQLite

## 核心架构原则

### 1. 跨平台优先
- 支持Windows、Linux、macOS三大平台
- 提供可执行文件、Python包、便携版本多种部署方式
- 避免平台特定依赖，优先使用标准库

### 2. 性能驱动
- 百万级词条处理为目标
- 数据库操作批量化和异步化
- 内存使用优化，避免全量加载

### 3. 扩展性设计
- 插件化架构，支持第三方功能扩展
- 模块化设计，服务间松耦合
- API优先，支持多种客户端

### 4. 用户体验至上
- 提供CLI、Web、GUI多种交互方式
- 实时反馈和进度提示
- 直观的错误信息和帮助文档

## 技术架构概要

### 总体架构
```
应用核心 (app/core/)
├── 运行模式管理器 (CLI/Web/GUI/API)
├── 服务层 (码表/统计/导入/插件)
└── 数据访问层 (SQLite/内存/文件适配器)

用户界面层 (ui/)
├── CLI (Typer + Rich)
└── GUI (Tkinter/PyQt)

数据存储层
├── SQLite数据库 (主存储)
├── 本地缓存文件
└── 用户配置文件
```

### 关键技术选择
- **API框架**: FastAPI - 高性能，自动文档，异步支持
- **CLI框架**: Typer - 现代CLI，代码即文档
- **数据库**: SQLite 3.40+ - 嵌入式，高性能
- **缓存**: diskcache - 本地文件缓存
- **打包**: PyInstaller + Nuitka - 多平台可执行文件

## 开发指南

### 代码质量标准
1. **类型安全**: 所有代码必须通过mypy严格模式检查
2. **代码风格**: 使用black格式化，isort排序导入
3. **代码检查**: 通过ruff检查，零警告
4. **测试覆盖**: 核心功能测试覆盖率 > 90%
5. **文档完整**: 所有公共API必须有文档字符串

### 项目结构规范
```
vm-tool/
├── app/                    # 应用程序核心
│   ├── core/              # 核心框架
│   ├── services/          # 业务服务
│   ├── dal/              # 数据访问层
│   ├── plugins/          # 插件系统
│   └── config.py         # 应用配置
├── ui/                    # 用户界面
│   ├── cli/              # 命令行界面
│   └── gui/              # 图形界面
├── tests/                 # 测试代码
├── docs/                  # 文档
├── data/                  # 数据文件
└── scripts/               # 构建脚本
```

- 加载100万词条: < 5秒
